## Rendering Mode

**Fully static export** — zero client-side JavaScript.

- Astro `output: 'static'` (default)
- No `client:*` directives anywhere — no islands
- `tastyIntegration({ islands: false })` — SSR style collection with no client JS shipped
- All pages prerendered at build time to plain HTML + CSS
- No hydration, no `<script>` tags in output

## Styling Approach

### Runtime `tasty()` only

All styling uses `tasty()` components and `useGlobalStyles()`. No `tastyStatic`.

- **`tasty()`** — creates styled React components used inside `.astro` pages
- **`useGlobalStyles()`** — injects global styles (resets, typography, body styles)
- **`useRawCSS()`** — injects raw CSS when needed (e.g., font imports)
- **`configure()`** — sets design tokens, state aliases, and typography presets

### Astro Integration

```ts
// astro.config.ts
import { defineConfig } from 'astro/config';
import react from '@astrojs/react';
import sitemap from '@astrojs/sitemap';
import { tastyIntegration } from '@tenphi/tasty/ssr/astro';

export default defineConfig({
  site: 'https://tenphi.me',
  output: 'static',
  integrations: [
    react(),
    tastyIntegration({ islands: false }),
    sitemap(),
  ],
});
```

### Color System — `@tenphi/glaze`

Colors are generated using `@tenphi/glaze`, an OKHSL-based color theme generator with automatic WCAG contrast solving. Glaze produces perceptually uniform light and dark schemes from a single hue/saturation seed. No manual color picking — contrast pairs are computed automatically.

**Theme file** (`src/config/theme.ts`):

```ts
// src/config/theme.ts
import { glaze } from '@tenphi/glaze';

const blue = glaze(210, 70);

blue.colors({
  surface: { lightness: 100, saturation: 0.08 },
  'surface-2': {
    base: 'surface',
    lightness: '-3',
    saturation: 0.12,
  },
  text: {
    base: 'surface',
    lightness: 0,
    contrast: 'AAA',
    saturation: 0.06,
  },
  'text-soft': {
    base: 'surface',
    lightness: 20,
    contrast: ['AA', 'AAA'],
    saturation: 0.04,
  },
  border: {
    base: 'surface',
    lightness: ['-12', '-22'],
    saturation: 0.3,
  },
  'accent-text': {
    base: 'surface',
    lightness: 50,
    contrast: ['AA', 'AAA'],
    saturation: 0.85,
  },
  'accent-text-hover': {
    base: 'surface',
    lightness: 40,
    contrast: ['AA', 'AAA'],
    saturation: 0.9,
  },
});

export const colorTokens = blue.tasty({
  states: {
    dark: '@dark',
  },
});
```

**How it works:**
- `glaze(210, 70)` — creates a blue theme with hue 210° and 70% saturation
- `.colors({...})` — defines semantic color roles with contrast constraints
- `contrast: 'AAA'` / `contrast: ['AA', 'AAA']` — Glaze solves for lightness that meets WCAG contrast ratios against the `base` color
- `lightness: ['-12', '-22']` — HCPair: first value for normal mode, second for high contrast
- `.tasty(...)` — exports tokens in Tasty's `{ '#name': { '': lightValue, '@dark': darkValue } }` format
- Dark mode colors are auto-generated by inverting and adapting the lightness curve

### Tasty Configuration

Call `configure()` in a shared module imported early by layouts. Color tokens come from Glaze.

```ts
// src/config/tasty.ts
import { configure } from '@tenphi/tasty';
import { colorTokens } from './theme';

configure({
  states: {
    '@mobile': '@media(w < 640px)',
    '@tablet': '@media(640px <= w < 1024px)',
    '@desktop': '@media(w >= 1024px)',
    '@dark': '@media(prefers-color-scheme: dark)',
  },
  tokens: {
    '$gap': '8px',
    '$radius': '6px',
    '$max-width': '720px',
    '$font': '"Inter", system-ui, -apple-system, sans-serif',
    '$monospace-font': '"JetBrains Mono", ui-monospace, monospace',
    ...colorTokens,
  },
  replaceTokens: {
    '$content-width': '$max-width',
  },
});
```

**Token mapping** (Glaze color names → Tasty token usage):

| Glaze color | Tasty token | Usage |
|---|---|---|
| `surface` | `#surface` | Page background, card fills |
| `surface-2` | `#surface-2` | Code blocks, subtle backgrounds |
| `text` | `#text` | Primary text |
| `text-soft` | `#text-soft` | Secondary text, metadata, captions |
| `border` | `#border` | Borders, dividers |
| `accent-text` | `#accent-text` | Links, accent text |
| `accent-text-hover` | `#accent-text-hover` | Link hover state |

## Design

### Principles

- **Minimalistic** — generous whitespace, limited palette, clean typography
- **Monochrome + one accent** — near-neutral surface with a single accent hue (blue, 210°) generated by Glaze
- **WCAG compliant** — all text/background pairs meet at least AA contrast (AAA where practical), enforced by Glaze's contrast solver
- **System-first typography** — Inter for body, monospace for code
- **Dark mode** — via `prefers-color-scheme` media query, no toggle (no JS). Dark scheme generated automatically by Glaze
- **Mobile-first responsive** — single column, fluid layout

### Typography Scale

| Preset | Size | Weight | Use |
|---|---|---|---|
| `h1` | 2rem / 1.2 | 700 | Page titles |
| `h2` | 1.5rem / 1.3 | 600 | Section headings |
| `h3` | 1.25rem / 1.4 | 600 | Subsection headings |
| `t1` | 1rem / 1.6 | 400 | Body text |
| `t2` | 0.875rem / 1.5 | 400 | Secondary text, metadata |
| `t3` | 0.75rem / 1.4 | 400 | Captions, timestamps |

Typography presets are defined via `configure({ replaceTokens })` or inline in components.

### Layout

- Max content width: `720px`
- Horizontal padding: `2x` (16px) on mobile, `3x` (24px) on tablet+
- Vertical rhythm: `gap` multiples for spacing

## Content Management

### Astro Content Collections

Blog posts use Astro's Content Layer API (v6) with Markdown files.

```
src/
  content.config.ts    ← collection schema with glob loader
  content/blog/
    hello-world.md     ← first post
```

**Schema:**
```ts
// src/content.config.ts
import { defineCollection, z } from 'astro:content';
import { glob } from 'astro/loaders';

const blog = defineCollection({
  loader: glob({ pattern: '**/*.md', base: './src/content/blog' }),
  schema: z.object({
    title: z.string(),
    description: z.string(),
    date: z.date(),
    draft: z.boolean().default(false),
  }),
});

export const collections = { blog };
```

**Frontmatter example:**
```yaml
---
title: "Hello World"
description: "First post on the new blog. A brief intro and what to expect."
date: 2026-04-07
---
```

## Global Styles

A `GlobalStyles` component uses `useGlobalStyles()` and `useRawCSS()` to set:

- CSS reset (margin, padding, box-sizing)
- `body` background and text color
- Base typography (font-family, line-height)
- `a` tag defaults
- Code block styling
- `img` max-width

```tsx
// src/components/GlobalStyles.tsx
import { useGlobalStyles, useRawCSS } from '@tenphi/tasty';

export default function GlobalStyles() {
  useRawCSS(`
    *, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }
    img { max-width: 100%; height: auto; display: block; }
  `);

  useGlobalStyles('html', {
    font: true,
    color: '#text',
    fill: '#surface',
    lineHeight: '1.6',
    '-webkit-font-smoothing': 'antialiased',
    '-moz-osx-font-smoothing': 'grayscale',
  });

  useGlobalStyles('code', {
    font: 'monospace',
    preset: 't2',
    fill: '#surface-2',
    padding: '0 .5x',
    radius: '.5r',
  });

  useGlobalStyles('pre', {
    fill: '#surface-2',
    padding: '2x',
    radius: '1r',
    overflow: 'auto',
  });

  useGlobalStyles('pre code', {
    fill: 'transparent',
    padding: '0',
  });

  return null;
}
```

## SEO

### Per-page metadata

Each page sets:
- `<title>` — unique per page, format: "Page Title — tenphi.me"
- `<meta name="description">` — unique description per page
- `<meta name="author" content="Andrey Yamanov">`

### Open Graph

- `og:title`, `og:description`, `og:url`, `og:type` (website/article)
- `og:site_name: "tenphi.me"`
- `og:locale: "en_US"`

### Twitter Card

- `twitter:card: "summary"`
- `twitter:title`, `twitter:description`

### Structured Data

JSON-LD in `<head>` for:
- **Home:** `Person` schema (name, url, sameAs links)
- **Blog post:** `BlogPosting` schema (headline, datePublished, author)

### Technical SEO

- `robots.txt` in `public/` — allow all
- `sitemap.xml` — auto-generated by `@astrojs/sitemap`
- Canonical URLs on all pages
- Semantic HTML: `<header>`, `<main>`, `<article>`, `<footer>`, `<nav>`, `<time>`
- `lang="en"` on `<html>`
- `<meta charset="utf-8">`
- `<meta name="viewport" content="width=device-width, initial-scale=1">`

## Fonts

Use system font stack by default. No custom font files for the initial version — keeps the build zero-dependency and fast.

```
font-family: "Inter", system-ui, -apple-system, "Segoe UI", Roboto, sans-serif;
```

Inter is referenced but falls back to system fonts if unavailable. A Google Fonts `<link>` for Inter can be added to `<head>` in the layout for better typography, or omitted entirely for maximum performance (system fonts only).

**Decision for v1:** Use a Google Fonts `<link>` for Inter (weight 400, 600, 700). This is a single HTTP request and avoids self-hosting font files.

## Markdown Styling

Styled elements inside blog post content:

- `ul`, `ol` — lists with left padding
